# TTS generation speedups (2026-05-30)

Goal: cut the wait for NEW Pokedex entries (custom + autogenerated fusions)
without touching quality or features. Repeat entries were already instant
(content-hash WAV cache in main.rb) and sentence streaming already plays the
first sentence while the rest generate. The remaining cost was raw GPU
inference, so the changes below attack that.

## What changed

1. **torch.compile is now ON by default on CUDA** (`server.py`).
   The single biggest win the engine offers: 2-3x faster inference on the
   RTX 4090. It was previously off because the inductor backend needs Triton,
   which has no stock Windows wheel.
   - `--no-compile` still disables it if anything goes wrong.
   - `torch._dynamo.config.suppress_errors = True`: if Triton/inductor is ever
     missing, compile silently falls back to eager (no speedup, but TTS keeps
     working - zero feature loss).
   - Pre-warm: when compiled, the startup warmup runs a longer second pass so
     the ~30-90s JIT cost is paid DURING server load (while you walk to the
     Pokedex), not on the first real entry. Kernels are cached in
     `.torch_cache/`, so every later launch is fast.

2. **Triton is installed automatically** (`setup.py` -> `install_triton`).
   On Windows it installs `triton-windows` (3.1.x, matched to torch 2.5.x from
   the cu121 channel). Best-effort: if it can't install, you just lose the
   compile speedup, nothing breaks.

3. **Free GPU math tuning** (`server.py`): TF32 matmul, cudnn.benchmark, and
   high float32-matmul precision. Speeds up the prefill/decoder matmuls on the
   4090 with no audible change. Active even with `--no-compile`.

## How to activate (one-time)

Triton is only installed by setup.py, so force a clean reinstall:

1. Delete BOTH `fish_speech/python/` AND `fish_speech/.installed`
   (deleting python/ alone makes the launcher skip setup.py).
2. Relaunch `Start_TTS_Server.bat`. First launch reinstalls deps (incl.
   triton-windows) and pays the one-time compile JIT during warmup.
3. Confirm in `fish_speech/server.log`:
   - `CUDA math tuning enabled ...`
   - `torch.compile requested - dynamo suppress_errors ON ...`
   - `Compiling function...` (from the engine) and a slower-then-fast
     `Background warmup pass 2 (compile specialisation) done in ...`
   If you instead see a dynamo/Triton traceback followed by normal output,
   compile fell back to eager - check that triton-windows installed.

## How to benchmark before/after

The server already logs each generation. To get a clean A/B number, run a
fixed entry twice (delete its cached WAV first so it regenerates):

  - Note `_generate_tts_fish_speech: success in N.NNs` in `debug.log`, OR
  - Hit the server directly with a fixed sentence and time it:

      curl -s -o out.wav -w "%{time_total}s\n" \
        --data "text=It uses its sturdy front legs to push through the densest forests." \
        http://127.0.0.1:7861/tts

Compare with `--no-compile` (slow baseline) vs default (compiled). On a 4090
expect roughly 2-3x faster steady-state generation, plus TF32 on top.
